# AI協働ワークフロー (WORKFLOW.md)

## 1. ブランチ命名規則
AIの支援を受けて機能開発を行う際は、以下の命名規則を使用してください：
`ai/[username]/[feature-name]`

**例:**
- `ai/ryu/fix-header-layout`
- `ai/tanaka/add-login-page`

## 2. セッション管理
- **進捗共有:** 30〜60分ごと、または大きなタスクが完了するたびに、AIに `STATUS.md` の更新または進捗の要約を求めてください。
- **引き継ぎ (セッション終了時):**
  セッションを終了する際、必ずAIに以下を指示してください：
  > 「今回のセッションで行った作業を要約し、残っているTODOをリストアップしてください。また、次のセッションで最初に行うべき手順を提案し、これらを `NEXT_STEP.md` に保存してください。」

## 3. 「3回ルール」
トークンの浪費とコードの品質低下を防ぐためのルールです：
- AIが**同じエラーで3回**解決に失敗した場合：
  1. 現在のアプローチを**中止（STOP）**してください。
  2. ユーザー（人間）に助言を求めるか、手動での修正を依頼してください。
  3. 盲目的に再試行を繰り返さないでください。
  4. セッションをリセットするか、最後に正しく動作していた状態まで戻すことを検討してください。

## 4. コードのオーナーシップ
- **リセットを恐れない:** AIが泥沼（ウサギの穴）にはまり込んだ場合、時間をかけてデバッグするよりも、`git checkout .` で変更を破棄し、より明確なプロンプトで最初からやり直す方が早いことがよくあります。
- **人間のレビューは必須:** AIのコードはあくまで「提案」です。メンテナはあなたです。コミットする前に必ず `git diff` で変更内容を確認してください。
