# 推奨プロンプト集 (PROMPTS.md)

このファイルは、AIエージェントに効果的な指示を出すためのテンプレートです。

## デイリーワークフロー

### 1. コンテキスト同期 (セッション開始時)
**目的:** AIに現在のプロジェクトの状態を把握させる。
```markdown
`git pull origin main` を実行して最新の変更を取り込んでください。
その後、リポジトリの構造（`tree -L 2` を実行）と、最新の `package.json` の依存関係を確認してください。
現在のプロジェクトの状態と、私が知っておくべき新しい依存関係があれば要約して報告してください。
```

### 2. アトミックなタスク (機能実装)
**目的:** AIに「1つのこと」だけに集中させる。
```markdown
[FEATURE_NAME] を `[FILE_PATH]` に実装してください。
コーディングを始める前に:
1. `src/lib` などにある既存のユーティリティで使えそうなものがないか確認してください。
2. 実装の手順を箇条書きで示してください。
3. 変更はこの特定の機能に限定し、最小限に留めてください。
```
**例:** 「ログインフォームのバリデーションを `src/components/auth/LoginForm.tsx` に実装してください。`zod` がインストールされていれば使い、なければ `src/lib/validation.ts` を確認してください。」

### 3. テスト駆動開発 (TDD)
**目的:** 実装の信頼性を担保する。
```markdown
[FEATURE] を実装したいです。
まず、この機能が正しく動作することを期待するテストケースを `__tests__/` (または適切なテストディレクトリ) に作成してください。
テストを実行し、失敗することを確認してください（Red）。
その後、テストが通るように機能を実装してください（Green）。
最後に、もう一度テストを実行し、成功した結果を見せてください。
```

### 4. 差分レビュー (コミット前)
**目的:** 意図しない変更を防ぐ。
```markdown
`git diff` を実行して、あなたが加えた変更を見せてください。
[FEATURE_NAME] とは関係のないファイルが変更されていないか特に注意して確認してください。
もし無関係な変更があれば、それらを元に戻してください。
```

## シナリオ別プロンプト

### バグ修正
```markdown
バグが報告されました: [BUG_DESCRIPTION]。
1. `grep` や検索ツールを使って、関連するコード箇所を特定してください。
2. バグを再現し、失敗するテストケースを作成してください。
3. バグの原因を「3行」で説明してください。
4. バグを修正し、テストが通ることを確認してください。
```

### 仕様変更 / リファクタリング
```markdown
[FEATURE] の動作を [NEW_BEHAVIOR] に変更したいです。
修正が必要になるファイルをリストアップしてください。
作業を始める前に、潜在的なリスクや既存機能への影響（デグレードのリスク）を特定して教えてください。
```
