# 動作確認ガイド (Pre-Deployment)

ローカル環境でWebアプリを起動し、各機能をテストする手順です。

## 1. 開発サーバーの起動

ターミナルで以下のコマンドを実行してください：

```bash
npm run dev
```

起動したら、ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスします。

---

## 2. テストシナリオ

### A. 来場者フロー (Visitor)
1. **トップページ**: 
   - [http://localhost:3000/](http://localhost:3000/) を開く。
   - 「企画一覧」をクリック。
2. **企画一覧**:
   - 検索やタブ切り替えが動作するか確認。
   - 任意の企画（例: 1-1）をクリック。
3. **企画詳細 & 投票**:
   - 「この企画に投票する」をクリック。
   - ログイン画面に飛ばされる -> 「アカウント登録はこちら」から適当なメアド/Passで登録。
   - 登録後、自動でログインされる（またはログインし直す）。
   - 再度「投票」をクリック -> 「投票しました」と出るか確認。
4. **整理券 (FastPass)**:
   - 「整理券を取得」ボタンがある企画を探す（`seed.sql`で `fastpass_enabled=true` の企画）。
   - 時間枠を選んで取得 -> 成功するとマイページへ。
5. **マイページ**:
   - QRコードが表示されているか確認。

### B. 運営者フロー (Operator)
*別のブラウザウィンドウ、またはシークレットウィンドウ推奨*

1. **ログイン**:
   - [http://localhost:3000/operator/login](http://localhost:3000/operator/login) にアクセス。
   - ID: `1-1` (または `seed.sql` にあるクラスID)
   - Pass: `password` (初期値)
2. **ダッシュボード**:
   - **混雑度更新**: 「混雑」などのボタンを押す -> **来場者側の企画詳細ページでアイコンが変わるか確認** (Realtime)。
   - **QRスキャン**:
     - 「QRコードをスキャン」をクリック（カメラ許可が必要）。
     - 来場者側のマイページで表示したQRコードをカメラにかざす。
     - 「確認OK」と出るか確認。
     - 同じQRをもう一度スキャン -> 「既に使用済み」等のエラーが出るか確認。

### C. 管理者フロー (Admin)
1. **ログイン**:
   - [http://localhost:3000/admin/login](http://localhost:3000/admin/login) にアクセス。
   - **重要**: 事前にSupabase上で、ログインしたいユーザーの `role` カラムを `admin` に変更しておく必要があります。（または `auth.users` のトリガーで作成された `public.users` のレコードを編集）
2. **ダッシュボード**:
   - 投票ランキングが表示されているか確認。
   - 「システムデータをリセット」はテストデータの消去に便利です。

---

## 3. 既知の注意点
- **Realtime機能**: ローカル環境でもSupabaseと接続されていれば動作しますが、ネットワーク環境によっては遅延することがあります。
- **QRスキャン**: PCにカメラがない場合、QR動作確認はできません（コード入力などの代替手段は現在未実装です）。
