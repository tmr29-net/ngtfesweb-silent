# フロントエンド仕様書

---

## 1. 概要

本ドキュメントは、文化祭Webサイトのフロントエンド実装（Next.js App Router）に関する仕様を定義する。
`仕様書.md` および `バックエンドAPI仕様書.md` に基づき、来場者・運営者・管理者が利用する画面構成、ユーザー体験（UI/UX）、および状態管理の方針を記述する。

---

## 2. 技術スタック・アーキテクチャ

* **Framework**: Next.js (App Router)
* **Language**: TypeScript
* **State Management**: React Context + Hooks (SWR/TanStack Query推奨)
* **UI Library**: Tailwind CSS (User requested) or CSS Modules
* **Deploy**: Vercel (Recommended)
* **Auth**:
  * **Visitor**: ID/Password Login (Supabase Auth)
  * **Operator**: Client-side Token Management (`operator_token`)

## 4. 共通UI/UX仕様

### 4.1 ヘッダー・フッター

* **Header**:
  * モバイルファースト設計（ハンバーガーメニュー）。
  * ログイン状態により「マイページ」アイコンを表示。
* **Footer**:
  * コピーライト、プライバシーポリシー、利用規約、運営者ログインへのリンク（目立たせない）。

### 4.2 状態管理・認証

* **Visitor (ID/Pass)**:
  * Supabase Auth Session を `SessionContext` で全体管理。
  * 未ログイン状態で「投票」「FP発券」を押すとログイン画面へ誘導。
* **Operator (Token)**:
  * `operator_login` RPC から返却された Token を `LocalStorage` または `SessionStorage` に保存。
  * `OperatorContext` を作成し、Tokenの有無でリダイレクト制御を行う。

### 4.3 エラーハンドリング

* **APIエラー**:
  * `401 Unauthorized`: ログイン画面へリダイレクト。
  * `429 Rate Limit`: トースト通知で「更新頻度が高すぎます。しばらくお待ちください」と表示。
  * `500 Server Error`: 「サーバーエラーが発生しました」等の汎用メッセージ。

## 5. 詳細画面仕様

### 5.1 トップページ (`/`)
* **First View**: 文化祭ロゴ＋開催日時。
* **Main Menu**: 大きなタイルボタンで「企画を探す」「長田検定」「マイページ」「スタッフ紹介」へ誘導。
* **News**: 運営からのお知らせ（緊急停止情報など）。

### 5.2 企画一覧 (`/projects`)
* **UI**:
  * タブ: [すべて] [食品] [クラス一覧] [ステージ]
  * カードリスト: サムネイル、タイトル、**混雑状況アイコン**、**推定待ち時間**。
    * **アイコン仕様**: 3段階の顔文字アイコンを使用する。
      * 🟢 **空き** (Green/Smile): 待ち時間なし〜少
      * 🟡 **普通** (Yellow/Neutral): ほどほど
      * 🔴 **混雑** (Red/Sad): 非常に混雑
    * **待ち時間**: `get_estimated_wait_time` RPCの結果（分）を表示。
* **Realtime**:
  * Supabase Realtime を購読し、`congestion` テーブルの更新があれば即座にアイコン色を変更する。

### 5.3 企画詳細 (`/projects/[id]`)
* **情報表示**: 画像、タイトル、説明文。
* **混雑状況**: 現在の混雑度と最終更新時間を表示。
* **Action Area**:
  * **投票ボタン**:
    * 未投票時: 「この企画に投票する」
    * 投票済時: 「投票済み（変更する）」
    * **Feature Toggle**: `voting_enabled=false` の場合はボタンを非表示、または「現在投票期間外です」と表示。
  * **FastPass**:
    * `fastpass_enabled=true` の場合のみ表示。
    * 「整理券を取得する」ボタン → 空き枠モーダル展開。
    * 残り枠なし: グレーアウト＋「配布終了」。

### 5.4 クイズプレイ (`/quiz/play`)
* **Feature Toggle**: `quiz_enabled=false` の場合はトップページで「現在開催していません」と表示し、プレイ画面への遷移をブロックする。
* **UI Flow**:
  1. 「スタート」押下 → 問題配列取得 (`get_quiz_questions` RPC)。
     * 取得データには正解の **ハッシュ値** が含まれる。
  2. 1問ずつ表示し、ローカルで正誤判定を行う。
     * ユーザー回答をハッシュ化し、サーバーからのハッシュ値と比較する。
  3. 全問終了後、クライアントサイドでスコアを計算。
  4. スコアと署名ハッシュを生成し、送信 (`submit_quiz_score` RPC) → 結果画面へ。
* **制約**:
  * ブラウザバック防止（警告ダイヤログ）。
  * リロード時はセッション破棄扱い（最初から）。

### 5.5 運営ダッシュボード (`/operator/dashboard`)
* **Header**: 「1年1組 様」表示。ログアウトボタン。
* **混雑状況更新**:
  * [空き][普通][混雑] の3ボタン（顔アイコン付き）。
  * 選択中の状態をハイライト。
  * **ガイドライン表示**: 各ボタンの下または横に、以下の目安を常時表示する。
    * 🟢 **空き** (LVL1): 待機列 **20%未満**
    * 🟡 **普通** (LVL2): 待機列 **20%以上 80%未満**
    * 🔴 **混雑** (LVL3): 待機列 **80%以上**
  * 30秒ロック中はボタンをDisabledにし、カウントダウン表示推奨。
* **QR読取**:
  * `react-qr-reader` 等を使用。
  * 読取成功 → `verify_and_use_ticket` RPC実行。
  * 結果モーダル:
    * **OK**: 緑背景＋「入場OK」＋大きな◯＋効果音。
    * **NG**: 赤背景＋「入場不可」＋理由（時間外、使用済）を表示。
* **情報編集 (画像・テキスト)**:
  * **画像アップロード**:
    * `react-easy-crop` 等を使用し、クライアントサイドで **アスペクト比 4:3** にトリミングする。
    * Canvas API を用いて長辺 1600px 以下、圧縮率 80% 程度にリサイズ・圧縮してから送信する。
    * 1クラスにつき1枚のみ（上書き保存）。
  * **テキスト**: 500文字制限。絵文字対応。

### 5.6 管理ダッシュボード (`/admin/dashboard`)
* **混雑状況一括管理**:
  * テーブル表示: [クラスID] [クラス名] [現在状況(変更btn)] [最終更新]
  * **ソート機能**: 混雑度順（混んでる順）、更新が古い順（放置検知用）。
* **ファストパス管理**:
  * 各クラスの「FP停止/再開」スイッチ。
  * 「全システム緊急停止」ボタン（赤色、二段階確認）。
* **操作ログ**:
  * 直近の管理者操作ログを表示する簡易ビューワ。

### 5.7 スタッフ紹介 (`/staff`)

* **UI構成**  
  1. **テーマ画像**：ページ上部にキービジュアルを表示（レスポンシブ対応）  
  2. **委員長挨拶**：氏名・役職名・挨拶本文を表示  
  3. **テーマ考案者コメント**：氏名・クラス・コメントを表示  
  4. **副テーマ考案者コメント**：氏名・クラス・コメントを表示  
 **データ取得**：静的生成 (SSG) による取得

## 6. 実装における注意点

### 6.1 画像最適化
* ユーザーがアップロードした画像は `next/image` を使用し、WebP形式で最適化して配信する。
* アップロード時はクライアントサイド（Canvas等）でリサイズ・圧縮してから送信する（通信量削減）。

### 6.3 ネットワーク異常への対応
* **基本方針**:
  * 「閲覧」はキャッシュを活用して可能な限り継続させる。
  * 「更新・重要な操作」は通信必須とし、失敗時は明確に機能をロックする。
* **表示仕様**:
  * **閲覧（一覧・マップ）**: 通信失敗時は LocalStorage/SWR キャッシュを表示し、画面上部に「オフラインモードで表示中（最新ではない可能性があります）」と警告バーを出す。
  * **更新不可**: 投票ボタン、FP発券ボタン等はグレーアウトし、「通信不安定のため利用不可」とツールチップ等で表示する。
  * **共通エラー文言**:
    > 現在ネットワークが不安定なため、一部機能を停止しています。
    > 情報閲覧は引き続き可能です。
